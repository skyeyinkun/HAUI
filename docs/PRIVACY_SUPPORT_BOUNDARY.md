# HAUI 隐私、售后与授权边界

## 隐私说明

- HAUI 默认运行在用户自己的 Home Assistant / Add-on 环境中。
- 设备列表、房间、人员、摄像头配置、AI Key、授权信息默认保存在用户自己的 HA Add-on 数据目录或浏览器本地存储。
- HAUI 不提供摄像头云转发服务。摄像头画面应优先走 Home Assistant 本地代理、go2rtc、RTSP 或设备厂商通道。
- AI 功能优先使用用户自己填写的 API Key。开发者不应默认代付或混用自己的 Key，除非单独签订服务规则。

## 售后边界

一次性部署费用建议仅覆盖：

- HAUI Add-on 安装。
- 首次连接和基础设备映射。
- 授权码生成和激活。
- 约定版本内的基础使用指导。

不默认覆盖：

- Home Assistant 本体损坏、升级失败或系统重装。
- 第三方集成、摄像头厂商平台、AI 服务商故障。
- 用户网络、路由器、NAT、SSL、域名、Nabu Casa 等环境问题。
- 用户自行修改配置导致的数据丢失。

## 授权和更新

- 系统部署后必须先完成授权，授权通过后开放完整控制界面。
- 授权码绑定当前 HAUI 机器码，换实例或重装后需要人工重新生成。
- 换机、迁移、重装建议人工处理，避免授权码被批量转卖。

## Home Assistant 关系声明

HAUI 是面向 Home Assistant 生态的第三方控制面板，不代表 Home Assistant 官方背书、认证或官方支持。

## 更新前建议

- 更新前先在 `系统设置 -> 备份恢复` 创建服务器备份，并下载一份本地备份。
- 远程部署时保留旧镜像或旧版本安装包，便于回滚。
- 摄像头、AI、授权相关变更应优先在测试环境验证。
