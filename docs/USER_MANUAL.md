## 快速开始

### 5.15.0 使用重点

HAUI 5.15.0 推荐按 **Home Assistant Add-on 优先，PWA 和平板墙屏兼容** 的方式使用：

- 普通用户从 Home Assistant 左侧栏进入 HAUI，不需要单独安装原生 App。
- 手机和平板可通过浏览器的 **添加到主屏幕** 方式作为 PWA 使用。
- 平板墙屏建议使用横屏、常亮、默认静音、摄像头隐私模式和备份恢复。
- AI、Agent、摄像头高级能力、墙屏/商业入口、备份恢复等 Pro 能力需要授权码。
- 系统部署后必须先完成授权，授权通过后开放完整控制界面。

### 安装方式

#### 方式一：Home Assistant Add-on（推荐）

1. 在 Home Assistant 中，进入 **设置** → **加载项** → **加载项商店**
2. 添加仓库地址：`https://github.com/skyeyinkun/HAUI`
3. 搜索并安装 **"HAUI - 智能家庭中枢"**
4. 按需填写 Add-on 配置项，例如授权公钥、AI 配置、萤石云 Key
5. 启动加载项，通过 Home Assistant 左侧栏或 **打开 Web UI** 使用

#### 方式二：PWA 安装到手机或平板

1. 先通过 Home Assistant Add-on 或独立服务打开 HAUI。
2. 使用 Safari、Chrome、Edge 等浏览器打开 HAUI 地址。
3. 在浏览器菜单中选择 **添加到主屏幕** 或 **安装应用**。
4. 从桌面图标进入后，HAUI 会以近似 App 的独立窗口运行。

说明：
- 当前 5.15.0 不包含 Android APK、iOS TestFlight 或 App Store 原生包。
- PWA 依赖浏览器和 Home Assistant 访问地址，离线后无法控制真实设备。

#### 方式三：独立部署 / 本地开发

```bash
# 克隆仓库
git clone https://github.com/skyeyinkun/HAUI.git
cd HAUI

# 安装依赖
npm install

# 构建项目
npm run build

# 本地开发预览
npm run dev
```

如果要按 Add-on 交付，请使用：

```bash
npm run build:addon
```

该命令会把前端产物复制到 `addon/dist`，用于 Home Assistant 本地加载项或 Docker 镜像交付。

### 首次配置

1. 打开 HAUI Dashboard 界面
2. 点击右下角 **设置** 按钮
3. 在 **Home Assistant 配置** 标签页中填写：
   - **本地地址**: 如 `http://homeassistant.local:8123`
   - **公网地址**（可选）: 如 `https://your-domain.ui.nabu.casa`
   - **访问令牌**: 从 HA 用户资料页面创建长期令牌
4. 点击 **保存配置**
5. 建议完成设备、房间、摄像头和 AI 设置后，进入 **备份恢复** 创建第一份备份

## 功能介绍

### 1. 智能仪表盘

#### 设备控制
- **开关控制**: 点击设备卡片即可开关
- **亮度调节**: 支持灯光亮度滑动调节（带防抖优化）
- **色温调节**: 支持冷暖色温调整
- **窗帘控制**: 支持开合度百分比调节

#### 场景模式
- 预设场景一键激活
- 支持自定义场景配置
- 场景执行状态反馈

### 2. AI 助手

#### 语音交互
- 点击右下角 AI 图标启动语音对话
- 支持自然语言控制设备
- 支持查询设备状态、统计传感器数据、查看天气信息
- 支持阿里云百炼、DeepSeek 官方 API、SiliconFlow 和自定义 OpenAI 兼容接口
- 在 HA Ingress 中优先走 Add-on 后端代理，减少浏览器直接暴露 API Key 的风险
- 控制灯光、窗帘、空调等低风险设备前会要求用户确认
- 门锁解锁、安防解除、重启、全域通配符控制等高风险操作会被阻止
- 支持 Agent 控制台，用于三通道模型配置、提案审批、Workspace 文档、长期记忆、心跳任务、工具状态和审计日志

#### 5.15.0 交付注意
- AI 设置保存会等待后端确认，保存失败会显示真实错误。
- Ingress 环境下优先使用 `/api/ai/chat` 后端代理，减少 API Key 暴露在浏览器端。
- 用户也可以在 Add-on 配置中填写 AI 参数，作为远程部署时的统一兜底配置。

#### 智能推荐
- 基于使用习惯的设备推荐
- 异常状态智能提醒

完整介绍、名词解释和使用步骤见 [HAUI AI 助手详细指南](./AI_ASSISTANT_GUIDE.md)。

### 3. 环境监控

#### 实时数据显示
- 温度、湿度、光照传感器
- 能耗统计与趋势分析
- 天气信息显示
- 摄像头监控组件支持 HA HLS、萤石云代理和 go2rtc

#### 区域管理
- 支持多房间/区域划分
- 区域设备快速筛选
- 常用设备快捷入口

### 4. 万能遥控

#### 红外遥控
- 支持电视、空调、风扇等设备
- 自定义红外码学习
- 遥控面板可视化

#### 气候控制
- 空调温度、模式、风速调节
- 定时开关设置
- 智能温控策略

## 操作指南

### 设备控制

#### 开关设备
1. 在主界面找到目标设备卡片
2. 点击卡片任意位置即可切换开关状态
3. 开关状态会实时同步到 Home Assistant

#### 调节灯光亮度
1. 长按灯光设备卡片进入详情
2. 拖动亮度滑块调节（0-100%）
3. 松开后自动保存（防抖优化，避免频繁请求）

#### 控制窗帘
1. 在窗帘卡片上拖动滑块调节开合度
2. 0% = 完全关闭，100% = 完全打开
3. 点击卡片可快速切换全开/全关

### 场景控制

#### 激活场景
1. 在主界面上方找到场景卡片
2. 点击场景图标即可激活
3. 场景激活后有 3 秒冷却时间

#### 编辑场景
1. 进入 **设置** → **场景配置**
2. 添加或修改场景名称
3. 配置场景对应的 HA 实体

### 区域管理

#### 切换区域
1. 点击顶部区域标签栏
2. 选择目标区域（客厅、卧室、厨房等）
3. 设备列表会自动筛选该区域设备

#### 设置常用设备
1. 长按仪表盘空白处进入编辑模式
2. 点击设备卡片上的星标图标
3. 常用设备会显示在"常用"标签页

### AI 语音控制

#### 启动语音助手
1. 点击右下角 AI 图标
2. 允许浏览器使用麦克风权限
3. 听到提示音后说出指令

#### 常用语音指令示例
- "打开客厅灯"
- "把卧室空调调到 26 度"
- "统计一下家里现在有哪些设备开着"
- "客厅温度和湿度是多少"
- "今天天气怎么样"

### 摄像头监控

#### 添加摄像头
1. 进入 **设置 -> 摄像头**。
2. 点击 **添加摄像头**。
3. 选择类型：
   - **HA (HLS)**：填写 HA 摄像头实体 ID，例如 `camera.front_door`，或填写 HA HLS URL。
   - **萤石 (Ezviz)**：填写萤石直播地址和 Token，或在 Add-on 配置中填写 AppKey/AppSecret 后测试。
   - **RTSP (go2rtc)**：填写 go2rtc 服务地址和流名称。
4. 点击测试按钮确认连接。

#### 隐私和墙屏
- 默认静音适合公共空间和平板墙屏。
- 开启隐私模式后，摄像头卡片默认不显示画面，需要点击 **显示画面**。
- HAUI 不提供摄像头云转发服务，优先使用 HA 本地代理、go2rtc 或厂商通道。
- PTZ 控制会校验摄像头实体和方向参数，失败时会显示 Home Assistant 返回的错误，便于排查权限或设备能力问题。

## 配置说明

### Home Assistant 连接配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 本地地址 | 局域网内 HA 地址 | `http://192.168.1.100:8123` |
| 公网地址 | 外网访问地址（可选） | `https://xxx.ui.nabu.casa` |
| 访问令牌 | HA 长期访问令牌 | `eyJ0eXAiOiJKV1QiLCJhbG...` |

### AI 配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| AI 提供商 | 选择 AI 服务提供商 | 阿里云百炼 |
| API 密钥 | 服务商提供的 API Key | - |
| 模型 | 使用的 AI 模型 | `qwen-plus` |
| 基础 URL | API 服务端点 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

支持的内置服务商：
- 阿里云百炼：默认 `qwen-plus`，支持设备查询、状态汇总和低风险控制。
- DeepSeek 官方 API：默认 `deepseek-chat`，支持 OpenAI 兼容接口和工具调用。
- SiliconFlow：可继续使用 `deepseek-ai/DeepSeek-V3` 等兼容模型。
- 自定义：填写任意 OpenAI 兼容 Base URL 和模型名。

在 Home Assistant Add-on 中，也可以在 Add-on 配置里填写 `AI_PROVIDER`、`AI_API_KEY`、`AI_MODEL` 和 `AI_BASE_URL`，作为后端代理兜底配置。

### 设备映射配置

在 **设置** → **设备映射** 中：
- 将 HAUI 设备与 HA 实体进行关联
- 支持自动发现和手动配置
- 支持批量导入导出

### 天气配置

1. 进入 **设置** → **天气配置**
2. 选择所在省份、城市、区县
3. 系统会自动获取坐标信息

### Add-on 配置项

| 配置项 | 说明 |
|--------|------|
| `HAUI_LICENSE_PUBLIC_KEY` | 授权公钥，默认已内置公钥，可按需覆盖 |
| `AI_PROVIDER` | 默认 AI 服务商 |
| `AI_API_KEY` | 后端 AI 代理兜底 Key |
| `AI_MODEL` | 默认 AI 模型 |
| `AI_BASE_URL` | OpenAI 兼容接口地址 |
| `EZVIZ_APP_KEY` | 萤石云 AppKey |
| `EZVIZ_APP_SECRET` | 萤石云 AppSecret |

## 安全功能

### 公网访问安全

#### PIN 码保护
- 当通过公网地址访问时，高危操作（开锁、解除安防等）需要输入 PIN 码确认
- 首次使用需设置 4-6 位数字 PIN 码
- PIN 码在本地加密存储

#### 高危操作列表
- 门锁控制
- 安防系统解除
- 车库门控制
- 报警器控制

### 数据安全

#### 本地加密
- 访问令牌使用本地加密/混淆存储
- PIN 码加密存储在浏览器本地
- HAUI 默认不提供云端服务，不上传用户摄像头画面、HA Token、AI Key 或家庭设备数据
- Ingress 中 AI 请求可走 Add-on 后端代理，降低浏览器端暴露 API Key 的风险

#### CSP 安全策略
- 启用内容安全策略防止 XSS 攻击
- 限制外部资源加载
- 仅允许同源脚本执行

## 授权和备份

### 离线授权

1. 进入 **设置 -> 授权**。
2. 复制机器码并发送给开发者。
3. 开发者使用私钥生成授权码。
4. 粘贴授权码并点击 **激活授权**。

说明：
- 机器码由 Add-on 后端生成并持久化，换浏览器或平板访问不会变化。
- 授权码绑定单个 Home Assistant 实例。
- 授权验证优先由 Add-on 后端完成，前端只保存必要状态用于显示。
- 系统未授权时只显示授权页面，不开放主界面。
- 离线授权不能做到 100% 防破解，只能提高转卖和白嫖成本。

### 备份恢复

进入 **设置 -> 备份恢复**：

- **创建备份**：在 Add-on 持久化目录生成服务器备份。
- **下载备份**：下载当前配置 JSON。
- **导入恢复**：导入备份文件并覆盖当前配置。
- 恢复前会自动生成回滚快照。

备份包含面板布局、设备映射、摄像头配置、AI 配置和授权状态。

建议：
- 每次远程更新或迁移前，先创建服务器备份并下载一份本地备份。
- 如果恢复后界面异常，可使用恢复前自动生成的回滚快照重新导入。

## 故障排除

### 无法连接到 Home Assistant

**现象**: 界面显示"未连接到 Home Assistant"

**解决方法**:
1. 检查 HA 地址是否正确
2. 确认访问令牌是否有效
3. 检查网络连接
4. 查看浏览器控制台网络请求

### 设备状态不同步

**现象**: 设备状态与实际不符

**解决方法**:
1. 检查设备映射配置
2. 刷新页面重新连接
3. 检查 HA 实体状态是否正常
4. 查看 WebSocket 连接状态

### AI 语音无响应

**现象**: 点击 AI 图标无反应或无法识别语音

**解决方法**:
1. 检查浏览器麦克风权限
2. 确认系统已完成授权
3. 确认 AI API 密钥、Base URL、模型名配置正确
4. 检查网络连接
5. 查看浏览器控制台或 Add-on 日志

### 授权码无法激活

**解决方法**:
1. 确认授权码绑定的机器码与授权页显示一致
2. 确认授权码由当前私钥生成
3. 如你更换过私钥，请同步更新 Add-on 配置中的 `HAUI_LICENSE_PUBLIC_KEY`

### 摄像头无法播放

**解决方法**:
1. HA 摄像头优先填写 `camera.xxx` 实体 ID
2. 萤石云检查 AppKey/AppSecret 或 Token
3. RTSP 先在 go2rtc 中确认流可用
4. 开启隐私模式时需要点击 **显示画面**

### 场景执行失败

**现象**: 点击场景无反应或部分设备未执行

**解决方法**:
1. 检查场景映射配置
2. 确认 HA 场景实体存在
3. 检查设备是否在线
4. 查看日志输出

### 性能问题

**现象**: 界面卡顿、响应慢

**解决方法**:
1. 启用"减少动画"模式（设置 → 外观）
2. 减少同时显示的设备数量
3. 检查浏览器性能
4. 关闭不必要的浏览器扩展
