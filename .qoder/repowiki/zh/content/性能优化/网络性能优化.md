## 简介
本文件聚焦于HAUI项目的网络性能优化，围绕以下目标展开：
- WebSocket连接优化：连接池与复用、断线重连策略、心跳检测与延迟评估
- Home Assistant API调用优化：请求合并、缓存策略、超时与降级处理
- 天气数据API优化：请求去重、数据预取、CDN与HTTPS回退
- 静态资源优化：Vite构建、代码分割与懒加载
- 网络监控与带宽优化：指标采集、代理与跨域处理

## 项目结构
本项目采用前端单页应用架构，核心网络层由WebSocket连接封装、天气服务适配器与缓存管理组成；构建阶段通过Vite进行优化与代理配置。

## 详细组件分析

### WebSocket连接优化
- 连接复用与键控：对同一URL+Token组合进行缓存，避免重复创建连接
- 最佳URL选择：并行探测本地与公网URL，优先返回可达实例
- 断线重连：监听断开事件，延迟重连；心跳检测评估延迟
- 可用性验证：HTTP短超时探测+WebSocket回退，规避CORS限制

```mermaid
flowchart TD
Start(["开始"]) --> BuildKey["生成连接键(url+token)"]
BuildKey --> SameKey{"是否相同键?"}
SameKey --> |是| ReturnConn["返回现有连接"]
SameKey --> |否| ClosePrev["关闭旧连接(如有)"]
ClosePrev --> CreateAuth["创建长令牌认证"]
CreateAuth --> CreateConn["创建WebSocket连接"]
CreateConn --> AddListeners["绑定ready/disconnected/reconnect-error"]
AddListeners --> Ready["连接就绪"]
Ready --> Heartbeat["每10秒发送ping并记录延迟"]
Heartbeat --> Disconnected{"断开事件?"}
Disconnected --> |是| DelayReconnect["延时重连(5秒)"]
Disconnected --> |否| Heartbeat
```

### Home Assistant API调用优化

### 天气数据API优化
- 提供商切换：通过工厂模式在Open-Meteo与OpenWeatherMap之间切换
- 指数退避重试：最多三次，间隔1s、2s，提升弱网鲁棒性
- 缓存与离线回退：命中缓存立即显示，后台刷新；失败时使用陈旧缓存
- HTTPS回退：在HTTP环境下自动回退到HTTP以兼容内网环境

```mermaid
flowchart TD
Enter(["useWeather入口"]) --> CheckCoord["检查坐标变化"]
CheckCoord --> TryCache{"首次加载且命中缓存?"}
TryCache --> |是| ShowCache["展示缓存并后台刷新"]
TryCache --> |否| SetLoading["设置loading"]
SetLoading --> CallAdapter["工厂获取适配器并请求"]
CallAdapter --> Retry{"请求失败?"}
Retry --> |是| Backoff["指数退避等待"]
Backoff --> CallAdapter
Retry --> |否| SaveCache["写入缓存"]
SaveCache --> Done(["完成"])
CallAdapter --> Fallback{"主提供商失败?"}
Fallback --> |是| SwitchOWM["切换到OpenWeatherMap"]
SwitchOWM --> SaveCache
Fallback --> |否| Done
Done --> Offline{"网络异常?"}
Offline --> |是| Stale["读取陈旧缓存并提示"]
Offline --> |否| Done
```

## 性能考量
- 连接层面
  - 使用连接键复用，避免频繁握手
  - 心跳检测与延迟上报，辅助网络质量评估
  - 断线重连延迟与事件监听，提升稳定性
- 数据层面
  - 缓存TTL与陈旧即用策略，降低重复请求
  - 指数退避与主备提供商切换，增强弱网韧性
- 构建与传输
  - 相对路径与外部依赖排除，减小包体
  - 开发代理支持WebSocket，简化调试
- 建议
  - 对高频REST接口增加请求合并与节流
  - 在CDN前置与反向代理上启用HTTP/2与压缩
  - 对图片与媒体资源采用懒加载与格式优化
