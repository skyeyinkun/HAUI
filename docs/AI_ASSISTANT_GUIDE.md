# HAUI AI 助手详细指南

## 适用版本

- **适用版本**: HAUI 5.15.16+
- **适用平台**: Home Assistant Add-on / 独立部署
- **核心入口**: HAUI 右下角 AI 助手按钮

---

## 1. AI 助手是什么

HAUI AI 助手是内置在 HAUI Dashboard 中的智能家居对话与控制入口。它不是单纯的聊天窗口，而是把大语言模型、Home Assistant 实体状态、服务调用、语音交互和 Agent 管理能力连接在一起，让用户可以用自然语言完成常见智能家居操作。

它当前由两层能力组成：

1. **基础 AI 助手**
   - 负责日常对话、设备状态查询、自然语言设备控制、语音输入、AI 回复朗读。
   - 适合说“打开客厅灯”“客厅温度多少”“家里有哪些设备开着”这类即时任务。

2. **Agent 控制台**
   - 负责更完整的 Agent 管理能力，包括三通道模型配置、Agent 对话、工具执行记录、提案审批、Workspace 文档、长期记忆、心跳任务和审计日志。
   - 适合做配置、排查、规划、记录和需要可追溯审批的任务。

---

## 2. AI 助手能做什么

### 2.1 日常设备控制

可以直接输入或语音说：

- “打开客厅灯”
- “关闭书房主灯”
- “打开窗帘”
- “启动扫地机器人”
- “关闭电视”

HAUI 会先在当前 Home Assistant 实体中查找匹配设备，再调用安全白名单内的服务。白名单内且目标明确的控制会直接执行，不再额外要求二次确认。控制结果会用短句反馈，例如“已打开书房主灯”。

### 2.2 设备状态查询

可以询问：

- “客厅温度多少？”
- “家里现在有哪些灯是开的？”
- “哪些设备不可用？”
- “有没有低电量传感器？”
- “卧室湿度和客厅湿度对比一下”

支持工具调用的模型会优先通过工具读取当前实体状态；不支持工具调用的模型会基于 HAUI 已注入的设备上下文回答。

### 2.3 状态汇总与建议

可以让助手汇总当前家庭状态：

- “统计一下家里开启中的设备”
- “帮我看看有没有异常设备”
- “根据现在的状态给我一些自动化建议”
- “看看哪些传感器最近可能需要关注”

### 2.4 语音交互

AI 助手支持：

- 语音输入。
- AI 回复自动朗读。
- 手动暂停、继续、停止朗读。
- 关闭自动朗读，避免每次回复都播报。

### 2.5 Agent 控制与管理

Agent 控制台支持：

- 配置 Primary AI、Backup AI、Summary AI。
- 查看 Agent 工具列表和可用状态。
- 使用 Agent 对话执行更复杂的多步骤任务。
- 查看工具 trace，确认每一步工具调用是否成功。
- 管理提案，避免 AI 直接写入高风险配置。
- 管理 Workspace 文档、长期记忆、心跳任务和审计日志。

## 3. 两个入口的区别

| 入口 | 适合场景 | 特点 |
|------|----------|------|
| 基础 AI 助手聊天 | 日常查询、即时控制、语音对话 | 快、轻量、适合高频操作 |
| Agent 控制台 | 多模型配置、提案、记忆、工具审计、复杂任务 | 更完整、可配置、可追溯 |

建议日常控制使用基础聊天；需要配置 Agent、查看执行过程、审批变更时进入 Agent 控制台。

## 4. 快速开始

### 4.1 配置 Home Assistant 连接

1. 打开 HAUI。
2. 进入右下角设置。
3. 配置 Home Assistant 地址和长期访问令牌。
4. 确认主界面设备状态能正常同步。

AI 助手控制设备时会复用 HAUI 主应用的 Home Assistant 服务调用链路，因此必须先保证 HAUI 本身能正常控制设备。

### 4.2 配置基础 AI 模型

1. 打开 AI 助手。
2. 点击模型配置按钮。
3. 选择 AI 提供商。
4. 填写 API Key、模型名和 Base URL。
5. 保存配置。

常见配置：

| 提供商 | Base URL 示例 | 模型示例 |
|--------|---------------|----------|
| 阿里云百炼 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| SiliconFlow | `https://api.siliconflow.cn/v1` | `deepseek-ai/DeepSeek-V3` |
| 自定义 OpenAI 兼容接口 | 按服务商填写 | 按服务商填写 |

### 4.3 配置 Agent 三通道模型

1. 打开 AI 助手。
2. 点击扳手图标进入 **Agent 控制台**。
3. 切换到 **模型** 页。
4. 配置：
   - **Primary AI**: 主模型。
   - **Backup AI**: 备用模型。
   - **Summary AI**: 总结模型。
5. 保存 Agent 配置。

三个模型可以使用同一个 API，也可以分别使用不同服务商。只要接口是 OpenAI Compatible 风格，并且 Base URL、API Key、模型名正确即可。

## 5. 基础 AI 助手使用指南

### 5.1 文本对话

1. 点击右下角 AI 图标。
2. 在输入框输入问题或指令。
3. 按发送按钮。
4. 等待流式回复或控制结果。

示例：

```text
客厅现在温度多少？
```

```text
打开书房主灯
```

```text
统计一下家里有哪些设备开着
```

### 5.2 语音控制

1. 打开 AI 助手。
2. 点击麦克风按钮。
3. 授权浏览器麦克风权限。
4. 说出指令。
5. 语音识别完成后自动发送。

建议使用清晰的设备名称，例如“书房主灯”“客厅窗帘”“卧室空调”。如果只说“打开灯”，系统可能会要求你说得更具体。

### 5.3 自动朗读开关

1. 打开 AI 助手。
2. 点击语音菜单。
3. 切换 **自动朗读 AI 回复**。

关闭后，AI 回复仍会显示在聊天窗口，但不会自动播放语音。正在播放的朗读会被停止。

### 5.4 快捷控制

HAUI 对常见开关类指令做了快捷控制优化。命中后会绕过冗长对话，直接查找设备并调用 Home Assistant 服务。

支持示例：

- “打开书房主灯”
- “请帮我打开书房里的主灯”
- “关闭客厅灯”
- “打开电视”
- “关闭加湿器”

快捷控制会做这些安全检查：

- Home Assistant 必须已连接。
- 必须找到明确的目标实体。
- 不允许 `all` 或通配符控制。
- 一次最多控制 5 个实体。
- 目标实体不能是 `unavailable` 或 `unknown`。
- 高风险域和高风险服务会被阻断。

## 6. Agent 控制台使用指南

进入方式：

1. 打开 AI 助手。
2. 点击顶部的扳手图标。
3. 进入 **Agent 控制台**。

### 6.1 对话页

用于和 Agent Kernel 对话。

可用开关：

- **使用 Agent 工具**: 开启后，Agent 可以调用后端工具读取状态、执行低风险控制或创建提案。

适合任务：

```text
检查家里有哪些不可用设备，并总结可能原因。
```

```text
帮我记录一条记忆：我平时晚上 11 点后希望客厅灯保持低亮度。
```

```text
帮我为概览页规划一张客厅温湿度卡片，生成提案。
```

### 6.2 模型页

用于配置 Agent 的三通道模型。

#### Primary AI

主模型。默认处理 Agent 对话和工具调用。建议选择稳定、工具调用能力较好的模型。

#### Backup AI

备用模型。Primary AI 请求失败、超时或返回无效结果时接管。可以和 Primary 使用同一个服务商，也可以换成另一个服务商。

#### Summary AI

总结模型。用于对多模型结果做归纳，输出更统一的最终答复。适合使用成本较低、总结能力稳定的模型。

#### 最大循环步数

限制 Agent 一轮任务最多进行多少次“思考-工具-再思考”的循环。数值越大，复杂任务能力越强，但耗时和 token 消耗也更高。

### 6.3 提案页

用于查看和审批 AI 创建的变更提案。

提案机制的目的：让 AI 可以提出修改建议，但不直接越权写入高风险配置。

常见提案：

- Workspace 文档变更。
- 仪表盘卡片变更。
- 自动化/脚本配置建议。

注意：当前自动化和脚本类提案会保留为审批记录，不会直接写 YAML。这样可以避免 AI 生成错误 YAML 导致 Home Assistant 配置报错。

### 6.4 文档页

用于查看和修改 Workspace 文档。

Workspace 文档是 Agent 的长期工作说明和人格配置。修改文档时会先生成提案，批准后才应用。

常见文档：

| 文档 | 作用 |
|------|------|
| `IDENTITY.md` | 助手名称、身份定位 |
| `SOUL.md` | 语气、风格、人格基调 |
| `USER.md` | 用户基本信息和偏好 |
| `MEMORY.md` | 长期偏好和重要背景 |
| `TOOLS.md` | 环境设备、工具使用说明 |
| `AGENTS.md` | 文件分工、操作约束 |
| `HEARTBEAT.md` | 定时跟进任务规则 |
| `BOOTSTRAP.md` | 首次初始化流程 |

### 6.5 记忆页

用于管理长期记忆。

适合记录：

- 用户偏好。
- 家庭设备习惯。
- 常用房间名称。
- 特殊规则。

示例：

```text
用户晚上睡觉时不希望自动朗读 AI 回复。
```

```text
书房主灯实际是 switch.wall_switch_4，但用户习惯叫它书房主灯。
```

### 6.6 心跳页

用于创建周期性任务。

适合任务：

- 定时检查设备状态。
- 定时提醒低电量设备。
- 定时汇总环境状态。
- 定时执行轻量级巡检。

创建时需要填写：

- 任务名称。
- 任务内容。
- 间隔分钟数。
- 是否启用。

### 6.7 工具页

用于查看 Agent 当前知道哪些工具，以及哪些工具已实现。

工具页可以帮助判断：

- Agent 是否能读取 Home Assistant 状态。
- Agent 是否能执行低风险控制。
- 哪些高风险能力被禁用。
- 当前工具数量和实现状态。

### 6.8 审计页

用于查看 Agent 相关操作日志。

审计日志记录：

- 谁发起了操作。
- 操作类型。
- 操作参数摘要。
- 执行结果。
- 时间。

这是排查问题和确认 AI 行为边界的重要入口。

## 7. 名词解释

### AI 助手

HAUI 内置的智能对话入口。它可以理解自然语言，并结合 Home Assistant 状态完成查询、控制和建议。

### Agent

带有工具调用、记忆、提案和任务循环能力的 AI 助手。普通 AI 更像“问答”，Agent 更像“能分步骤执行任务的操作员”。

### Agent Kernel

HAUI 后端的 Agent 运行核心，负责接收对话、组织上下文、调度模型、执行工具、返回结果。

### Primary AI

主模型。默认处理 Agent 对话请求。

### Backup AI

备用模型。主模型失败时接管，提高可用性。

### Summary AI

总结模型。用于把多个模型或多个步骤的结果整理为最终回复。

### Base URL

AI 服务的接口地址。OpenAI 兼容接口通常以 `/v1` 结尾。

### API Key

访问 AI 服务的密钥。HAUI 会在 Agent 配置返回时脱敏显示，避免明文泄露。

### Model

模型名称，例如 `qwen-plus`、`deepseek-chat`。模型名必须和服务商支持的名称一致。

### OpenAI Compatible

指服务商提供的接口兼容 OpenAI 的请求格式。很多模型平台都支持这种格式，HAUI 因此可以接入多个服务商。

### Tool Use / Function Calling

模型调用工具的能力。比如 AI 不只是回答“我会打开灯”，而是生成一个工具调用，由 HAUI 安全执行 `light.turn_on`。

### Tool Trace

工具调用轨迹。用于显示 Agent 调用了哪些工具、是否成功、哪里失败。

### Entity

Home Assistant 中的实体，例如 `light.living_room`、`sensor.temperature`。每个实体代表一个设备、传感器或虚拟对象。

### Entity ID

实体唯一标识，例如 `switch.wall_switch_4`。用户不需要记住它，但 AI 和工具调用需要用它精确定位设备。

### Domain

实体域名，也就是 entity_id 点号前面的部分。例如 `light`、`switch`、`sensor`、`climate`。

### Service

Home Assistant 服务动作，例如 `turn_on`、`turn_off`、`set_temperature`。

### Service Call

服务调用。由 domain、service 和 service_data 组成，例如控制灯光：

```json
{
  "domain": "light",
  "service": "turn_on",
  "service_data": {
    "entity_id": "light.living_room"
  }
}
```

### Context

AI 当前能看到的上下文，包括对话历史、设备摘要、实体状态、Workspace 文档和相关记忆。

### Workspace 文档

Agent 的可编辑工作说明文档，用 Markdown 保存。它们决定 Agent 的身份、语气、记忆和工具使用规则。

### Memory

长期记忆。用于保存跨会话的偏好和事实。

### Proposal

提案。AI 对配置、文档或仪表盘的修改建议。提案需要用户审批，避免 AI 直接写入高风险内容。

### Audit

审计日志。记录 Agent 的工具调用、配置变更、提案审批等操作。

### Heartbeat

心跳任务。周期性执行的 Agent 任务定义，适合定时巡检或提醒。

### Streaming

流式输出。AI 回复会逐字或分段显示，不必等完整回答生成后才出现。

### STT

Speech To Text，语音转文字。用于语音输入。

### TTS

Text To Speech，文字转语音。用于朗读 AI 回复。

## 8. 安全边界

HAUI AI 助手默认遵循安全优先原则。

### 8.1 允许的低风险控制

常见允许范围：

- 灯光开关。
- 普通开关。
- 窗帘打开/关闭/停止/位置设置。
- 风扇控制。
- 媒体播放器播放、暂停、音量。
- 空调温度和模式。
- 加湿器。
- 扫地机器人启动、暂停、回充。

### 8.2 默认阻断的高风险操作

默认不允许 AI 直接执行：

- 门锁解锁。
- 安防系统解除。
- Home Assistant 重启或停止。
- 配置重载。
- 全域控制。
- 通配符控制。
- 不明确目标设备的批量控制。

### 8.3 提案优先

对自动化、仪表盘、Workspace 等配置写入类操作，优先走提案机制。脚本执行等白名单内且目标明确的控制会直接执行。

## 9. 常见问题

### 三个模型必须用同一个 API 吗？

不是。Primary AI、Backup AI、Summary AI 可以共用一个 API Key，也可以分别使用不同平台。

常见组合：

- Primary AI 用高质量模型。
- Backup AI 用另一个稳定服务商，防止主服务不可用。
- Summary AI 用成本更低但总结能力稳定的模型。

### 为什么 AI 有时说不能直接控制？

常见原因：

- Home Assistant 未连接。
- 模型不支持工具调用。
- 目标设备不明确。
- 设备当前不可用。
- 操作属于高风险域或高风险服务。

### 为什么“打开灯”会要求说具体名称？

因为家里可能有多个灯。为了避免误开误关，AI 需要明确目标，例如“打开客厅灯”或“打开书房主灯”。

### Agent 控制台和普通模型配置有什么区别？

普通模型配置服务于基础 AI 助手聊天。Agent 控制台里的模型配置服务于 Agent Kernel，包括工具循环、提案、记忆和审计。

### 为什么自动化/脚本没有直接写入 YAML？

这是刻意的安全设计。直接写 YAML 容易产生格式错误或重复配置，导致 Home Assistant 报错。HAUI 当前更倾向于对配置写入生成提案，由用户确认和进一步处理。

### 为什么发布后看不到 Agent 控制台？

请确认：

1. 已更新到 HAUI 5.15.16 或更高版本。
2. 浏览器缓存已刷新。
3. AI 助手窗口顶部是否出现扳手图标。
4. Home Assistant 自定义集成 `yinkun_ui` 已加载 Agent API。
5. 当前用户具备管理员权限。

## 10. 推荐使用方式

### 日常控制

使用基础 AI 助手：

```text
打开客厅灯
关闭书房主灯
客厅温度多少
```

### 复杂排查

使用 Agent 控制台：

```text
检查不可用设备，按区域归类并给出处理建议。
```

### 长期偏好

使用记忆页或 Agent 对话：

```text
记住：晚上 11 点后不要自动朗读长回复。
```

### 配置变更

使用提案机制：

```text
帮我规划一个客厅温湿度趋势卡片，先生成提案。
```

## 11. 故障排查速查

| 问题 | 可能原因 | 处理方式 |
|------|----------|----------|
| AI 无法回复 | API Key、Base URL 或模型名错误 | 检查模型配置 |
| 设备无法控制 | HA 未连接或目标实体不可用 | 检查 HAUI 主界面设备状态 |
| 回复很慢 | 模型响应慢或工具循环过多 | 降低最大循环步数或更换模型 |
| 没有工具 trace | 未开启“使用 Agent 工具” | 在 Agent 对话页开启工具 |
| 提案无法批准 | 用户不是管理员或提案状态不是 pending | 使用管理员账号并刷新提案列表 |
| 语音无响应 | 浏览器麦克风权限未授权 | 检查浏览器权限 |
| 自动朗读太打扰 | 自动朗读开关开启 | 在语音菜单中关闭 |

---

## 12. 一句话理解

HAUI AI 助手负责“自然语言控制和查询”，Agent 控制台负责“多模型调度、工具执行、提案审批、记忆和审计”。日常使用聊天入口，复杂任务和配置管理进入 Agent 控制台。
